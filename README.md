# 體感硬核健身房 🏋️‍♂️💥

> 以**使用者養成**為核心的體感健身遊戲：用真實動作（深蹲／伏地挺身／舉手）打怪，
> 所有訓練成果累積到「你的角色」身上，化為一個會成長、會預測未來的**數位分身**。
>
> 技術與原理細節請見 [FEATURES.md](FEATURES.md)。

---

## 一、遊戲介紹

你站在攝影機前，畫面上的怪物從四面八方湧來。想反擊，就得做出**標準的健身動作**——
深蹲一下發射雷射、伏地挺身一下三連射。動作越標準、做得越多，怪打得越乾淨，
而這些汗水不會白流：每一場訓練都會記到「你的使用者」身上，餵養一個由真實數據驅動的
**數位分身**——它會隨你的累計次數長出力量、耐力與體態，還會依你的 BMI 走勢預測「一個月後的你」。

- **核心玩法**：體感打怪。MediaPipe 即時偵測 33 點骨架，判定動作達標就自動瞄準發射雷射。
- **養成主軸**：所有數據（身高體重、BMI、歷史場次、分身屬性）都掛在某位使用者身上，多人可並存切換。
- **運動防護價值**：即時姿勢回饋＋動作不標準時的警訊音，兼具「AI 動作校正」的實用性。

### 核心動線

```text
選擇/創建使用者 → 主頁(HUB) → 訓練前設定 → 訓練(打怪) → 結算(看分身成長) → 回主頁
```

整個流程用 **Scene 狀態機**組織，每個畫面是一個 Scene，畫面之間流轉清楚。

### 怪物與挑戰模式

| 怪物 | 特性 |
|------|------|
| 🧟 殭屍 | 標準怪物，耐打、不會飛 |
| 💀 骷髏 | 中速怪物，血量適中 |
| 🦇 蝙蝠 | 飛行怪物，移動較快 |

挑戰模式：殭屍 / 骷髏 / 蝙蝠 / 混合（決定該場怪物的組成權重）。

## 二、安裝

### 系統需求

- Python 3.10+
- 網路攝影機（USB／內建），或用手機當攝影機（見「手機串流」）
- 第三方依賴：OpenCV、MediaPipe、NumPy、Pillow、pygame；資料儲存用標準庫 `sqlite3`（免安裝）。

### 安裝步驟

```bash
# 1. 安裝依賴
pip install -r requirements.txt

# 2. 執行（本機鏡頭）
python main.py
```

- 第一次執行會**自動下載**姿態偵測模型（lite 模型，約 6MB）到專案目錄。
- 第一次執行會在專案根目錄建立資料庫 `gym.db`（你的存檔）。
- 若系統缺中文字型，遊戲內文字仍可運作（會嘗試多個跨平台字型路徑）。

## 三、使用說明（操作）

### 1. 選擇使用者（USER_SELECT）
啟動後的第一個畫面。

| 按鍵 | 功能 |
|------|------|
| 數字鍵 | 選擇既有使用者 |
| `N` | 創建新使用者（依序輸入名稱、身高、體重） |
| `Q` / `ESC` | 離開 |

> 名稱以**英數字**為主：OpenCV 視窗無法使用中文輸入法。

### 2. 主頁（HUB）
顯示你的數位分身與等級、BMI 現值與本月變化、累計訓練摘要。

| 按鍵 | 功能 |
|------|------|
| `G` | 開始訓練 |
| `T` | 查看趨勢圖（含預測） |
| `D` | 查看數位分身詳情 |
| `B` | 更新身高體重（重算 BMI） |
| `U` | 切換使用者 |
| `Q` | 離開 |

### 3. 訓練前設定（PRE_GAME）
**每輪只練一個動作**，全部用數字鍵操作。

| 按鍵 | 功能 |
|------|------|
| `1` / `2` / `3` | 選擇本場動作（深蹲／伏地挺身／舉手，單選） |
| `4` / `5` / `6` / `7` | 選擇挑戰模式（殭屍／骷髏／蝙蝠／混合） |
| `ENTER` | 開始訓練 |
| `ESC` | 返回主頁 |

各動作的觸發條件與攻擊型態：

| 動作 | 觸發 | 攻擊 |
|------|------|------|
| 深蹲 | 站立，膝蓋角度 >160°→<110°→回升 | 雷射單體（最近 1 隻） |
| 伏地挺身 | 水平，肘部角度 >155°→<90°→回升 | 雷射齊射（最近 3 隻） |
| 舉手（測試） | 坐姿，手腕舉過肩再放下 | 雷射單體（方便驗證流程） |

### 4. 訓練中（PLAYING）

| 按鍵 | 功能 |
|------|------|
| `P` / `ESC` | 暫停 |
| `M` | 怪物圖鑑（記錄本場所有出現過的怪物、擊敗數與擊敗率） |
| `S` | 截圖（存成 `fitness_battle_時間.png`） |
| `F` | 測試發射：不用做動作直接射一發雷射（場上沒怪會自動生一隻，方便確認效果） |

遊戲機制：怪物從四邊生成並追蹤你，碰到你扣一條命（共 3 條）；分數每滿 100 升一級，
怪物隨等級變多變快。三條命用完即 Game Over，停留數秒後進入結算。

暫停（PAUSED）：按 `C` 續玩，或 `H`／`ESC` 放棄返回主頁（放棄的這場**不計入**紀錄）。

### 5. 結算（RESULT）
顯示本場各動作次數、分數、到達等級、消耗卡路里，寫入該使用者的紀錄，
並視覺化分身的成長量（如「力量 +20、升到 Lv.2」）。按 `ENTER`／`H` 返回主頁。

## 四、手機串流當攝影機（選用）

把鏡頭架遠以拍到全身（做伏地挺身時必要）。手機端安裝 IP 攝影機 App：

- **Android**：IP Webcam（`http://手機IP:8080/video`）或 DroidCam（`http://手機IP:4747/video`）。
- 確認手機與電腦在**同一個 Wi-Fi**。

來源指定方式（不需改程式碼，優先序：啟動參數 > 環境變數 > 設定檔 > 預設本機）：

```bash
python main.py --source http://192.168.0.10:8080/video   # 1) 啟動參數
GYM_CAMERA=http://192.168.0.10:8080/video python main.py # 2) 環境變數
# 3) 複製 camera.cfg.example 為 camera.cfg，於第一行填入來源
```

支援來源：本機鏡頭（`0`）、Wi-Fi 串流（`http`/`rtsp` URL）、USB 有線（`usb:1`）。
連線失敗會印出清楚錯誤並**自動退回本機鏡頭**；連本機都沒有時仍可瀏覽選單。
**不支援藍牙**（頻寬不足以承載即時視訊串流，請用 Wi-Fi 或 USB）。

## 五、疑難排解

| 狀況 | 處理 |
|------|------|
| 開不了攝影機 | 確認裝置已連接、未被其他程式占用；或用 `--source` 指定來源 |
| 動作數不到 | 確保**全身（含關節）入鏡**、光線充足；深蹲建議側面或正面、伏地挺身把鏡頭架遠側放 |
| 中文顯示異常 | 安裝任一 CJK 字型（如 Noto Sans CJK / 文泉驛微米黑） |
| 沒有提示音 | 確認已安裝 pygame 且 `assets/sounds/` 有音效檔；否則自動降級為系統嗶聲 |
| 想重置存檔 | 刪除專案根目錄的 `gym.db`（會在下次啟動重新建立） |
